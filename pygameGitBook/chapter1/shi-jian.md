# 事件

---

> 事件便是操控整个游戏的主要部分，像我们的鼠标点击，键盘输入这些操作都需要事件进行检测，然后做出对应的动作。比如当我按下鼠标，就能让图像移动。

#### 

#### event模块中的整体事件

* event模块的常用各种处理方法

| 方法名 | 功能 |
| :--- | :--- |
| pygame.event.get\(\) | 获取一个事件列表，配合for循环 |
| pygame.event.pump\(\) | 自动处理事件 |

* 用法：

```python
e = pygame.event.get()    # 获取一个整体事件
pygame.event.pump()       # 自动 处理
for i in e:    # 循环遍历事件
    # 如果这个类型等于 QUIT这个事件
    if i.type == pygame.QUIT:   
        # 执行退出代码
        pygame.quit()
        sys.exit()
```

* 例子：当鼠标移动的时候，打印一句话

```python
for i in e:
    if i.type == pygame.MOUSEMOTION:
        print("鼠标正在移动...")
```

* 常用的事件，都是通过pygame调用，如pygame.QUIT

| 类型 | 功能 |
| :--- | :--- |
| QUIT | 点击窗口的关闭按钮时 |
| KEYDOWN | 键盘按下 |
| KEYUP | 键盘松开 |
| MOUSEMOTION | 鼠标移动 |
| MOUSEBUTTONDOWN | 鼠标按下 |
| MOUSEBUTTONUP | 鼠标松开 |
| USEREVENT | 用户事件 |

if k[pygame.K_a]:

#### 用户事件

* 用户事件可以用于每隔多少秒就做某件事，比如，隔30秒出现敌军boss

* pygame库调用USEREVENT

* 如果创建多个用户事件，需要加或者加2等，如：pygame.USEREVENT + 1

* 用户事件搭配定时器使用

```python
BOSS_EVENT = pygame.USEREVENT    # Boss的用户事件
HERO_EVENT = pygame.USEREVENT + 1   # 英雄的用户事件
ENEMY_EVENT = pygame.USEREVENT + 2  # 敌人的用户事件
```
